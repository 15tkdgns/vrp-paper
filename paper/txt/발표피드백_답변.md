## 피드백 1. 모델 학습 과정의 명확화

**피드백 요지:** 학습 절차, 손실 함수, 하이퍼파라미터 설정 등에 대한 구체적인 설명이 부족함.

### 답변

본 연구의 학습 절차는 다음의 단계로 구성됩니다.

**손실 함수:** 모든 모델은 MSE(Mean Squared Error)를 최소화하는 방향으로 학습됩니다. 타깃은 전방 h-거래일 로그 실현분산(log-annualized forward RV)이며, 수식은 아래와 같습니다.

```
Target_i,t(h) = log( mean(r²_{t+1, ..., t+h}) × 252 + ε )
```

**학습 절차 요약:**

| 단계 | 내용 |
|------|------|
| 1. 데이터 분할 | 전체 기간의 전반부 80% → 학습세트, 후반부 20% → 테스트세트 (날짜순 고정 분할) |
| 2. Purge gap | 학습-테스트 경계에 h-거래일 gap 적용 (타깃 기간 중첩 차단) |
| 3. 내부 검증 (Inner holdout) | 학습세트 내 전반부 80% → 내부 학습, 후반부 20% → 검증 (동일한 purge gap 적용) |
| 4. 자산군별 독립 피팅 | Equity(5종) / Bond(3종) / Commodity(3종) 각각 독립적으로 HP 탐색 및 최종 모델 학습 |
| 5. 하이퍼파라미터 선정 | 내부 검증 R² 최대화 기준으로 최적 HP 선정 (테스트셋 정보 불사용) |
| 6. 최종 학습 | 선정된 HP로 전체 학습세트(inner 학습 + inner 검증)에서 최종 모델 재학습 |
| 7. 평가 | 테스트세트에서 Pooled R², Median R², RMSE 산출 |

**표준화:** 모든 피처에 Z-score 정규화 적용. 스케일링 파라미터(평균·표준편차)는 내부 학습구간에서만 추정하여 테스트 구간에 적용함으로써 데이터 누출(leakage) 방지.

**무작위 시드:** RF, XGBoost, LightGBM, BiLSTM-A, MLP는 단일 시드 `seed=42` 고정으로 재현성 확보.

**XGBoost 고정 파라미터:** `n_estimators=200`, `subsample=0.8`, `colsample_bytree=0.8`, `reg_alpha=1.0`, `reg_lambda=2.0`, `min_child_weight=5` (고정, 탐색 대상 아님). 탐색 대상은 `max_depth ∈ {3,4}`, `learning_rate ∈ {0.03, 0.05, 0.1}`.

## 피드백 2. Feature Engineering 절차

**피드백 요지:** 37개 → 35개 피처 선택 기준이 모호하며, Feature 소거 방식이 직관적으로 이해되지 않음.

### 답변

**피처 설계 방향:**
피처는 도메인 지식 기반으로 4개 그룹으로 설계하였습니다.

| 그룹 | 피처 수 | 설계 근거 |
|------|--------|---------|
| HAR 기반 (지연 RV, Vol-of-Vol, 모멘텀) | 12개 | Corsi (2009) HAR-RV 이론 |
| HF Proxy (Parkinson, GK, RS, Range 비율) | 7개 | 일별 가격 범위로 고빈도 정보 근사 |
| IV Surface (VIX 기간구조, VRP 대리변수) | 10개 | Carr & Wu (2009), 변동성 위험프리미엄 |
| Alternative (Amihud, Kyle λ, 거래량 지표) | 6개 | 시장 미시구조 기반 유동성·시장충격 |
| **합계** | **35개** | |

**37개 → 35개 제거 과정 (2개 제거):**

초기 설계에서 GARCH(1,1) 조건부 분산과 GARCH 잔차 제곱을 피처로 추가한 37개 후보군을 검토하였습니다. 그러나 GARCH 기반 피처 2개가 포함될 경우 Ridge 모델의 내부 검증 기반 정규화 강도 선택 과정에서 과적합 신호가 혼입되어 22거래일 Pooled R²가 약 0.006 하락하는 것이 확인되었습니다. 이는 GARCH 피처가 타깃과의 정보 경로 상 불필요한 잡음을 도입하기 때문입니다. 이에 따라 성능 검증 결과에 근거하여 GARCH 피처 2개를 제외하고 35개 피처를 최종 사용하였습니다.

**피처 소거 방식 직관:**
"더 많은 피처 = 더 좋은 성능"이 아닙니다. 피처를 추가할 때 내부 검증 성능이 오히려 하락한 경우 해당 피처를 제거하는 backward elimination 방식의 도메인 검증을 수행하였습니다.

## 피드백 3. 모델 선택의 정당성 (시계열 vs. XGBoost)

**피드백 요지:** 데이터가 시계열임에도 XGBoost를 채택한 근거를 명확히 해야 함. 피처 자체에 시계열 특성이 반영되어 있다는 점을 명시적으로 서술할 필요.

### 답변

**핵심 논지:** 본 연구는 시계열적 특성을 모델 아키텍처가 아닌 **피처 공학**을 통해 내재화합니다.

35개 피처 중 19개(HAR 기반 12개 + HF Proxy 중 rolling 기반 7개)가 명시적으로 시계열 lag, rolling window를 포함합니다.

| 시계열 정보 | 피처 예시 |
|------------|---------|
| 1·5·10·22거래일 지연 로그 RV | LogRV_lag1, LogRV_lag5, LogRV_lag10, LogRV_lag22 |
| 5·22거래일 롤링 표준편차 | LogRV_Std5, LogRV_Std22 |
| 변동성 모멘텀 | RV_Mom5, RV_Mom22 |
| 22거래일 롤링 추정량 | Parkinson_22, GarmanKlass_22, RogersSatchell_22 |
| VIX 5일 이동평균·변화율 | IV_VIX_ma5, IV_VIX_chg |
| 22거래일 VRP 이동평균 | IV_VRP_ma22 |

이러한 설계 하에서 XGBoost는 "시계열 모델이 아닌" 것이 아니라, 시계열 정보가 이미 인코딩된 피처를 입력으로 받아 비선형 상호작용을 포착하는 **크로스-섹션 회귀 모델**로 작동합니다. 이는 Gu, Kelly & Xiu (2020)가 금융 수익률 예측에서 채택한 접근법과 동일한 패러다임입니다.

**추가 근거:**
- 본 연구의 비교군인 HAR-3 자체가 계량경제학적 시계열 모형(Corsi 2009)이며, WEns는 이를 유의하게 상회(22d DM = +5.391, p < 0.001)합니다.
- BiLSTM-A(시계열 아키텍처)를 명시적으로 비교군에 포함하였으나, BiLSTM-A(0.5098) < WEns(0.7974)로 나타나 시계열 아키텍처의 우위가 없음을 실증적으로 확인하였습니다.

## 피드백 4. 데이터 구성 및 학습 절차의 재현성

**피드백 요지:** 학습/테스트 데이터 구성, 자산군별 다수 모델 학습 방식에 대한 기술이 부족하여 재현성 확보가 어려움.

### 답변

**데이터 구성:**

```
전체 표본: 2010.01 ~ 2024.12 (약 15년)
자산: 11개 ETF (Equity 5, Bond 3, Commodity 3)
패널 구조: 날짜 × 자산 (각 날짜에 11개 행)
학습세트: 전체의 80% (날짜순)
테스트세트: 전체의 20% (날짜순)
Purge gap: 학습-테스트 경계에 h-거래일 제외
```

**자산군별 다수 모델 학습 구조:**

각 예측 기간 h에 대해, 자산군(Equity / Bond / Commodity)별로 독립적인 모델을 학습합니다. 예를 들어 XGBoost의 경우:

```
XGBoost_Equity  ← Equity 5종(SPY, QQQ, IWM, EFA, EEM) 학습
XGBoost_Bond    ← Bond 3종(TLT, IEF, AGG) 학습
XGBoost_Commodity ← Commodity 3종(GLD, SLV, USO) 학습
```

이렇게 기간 8개 × 자산군 3개 = **24개 XGBoost 모델**이 학습되며, Ridge도 동일 구조로 24개가 학습됩니다. WEns의 경우 Ridge + XGBoost 가중치 pw까지 포함하여 기간당 3개(자산군) × 6개(pw 후보) = 18회 내부 검증 후 최적 pw 선정.

**재현성을 위한 고정값:**
- 랜덤 시드: `seed=42` 전 모델 통일
- 데이터 분할: 고정 비율(80/20), 날짜순 정렬 후 positional split
- 코드 저장소: 전체 실험 코드 `run_main_benchmark_v6.py`에 일원화하여 관리

## 피드백 5. 예측 기간(horizon)에 따른 성능 해석

**피드백 요지:** 1일 후 예측보다 252일 후 예측 Pooled R²가 더 높은 현상은 직관에 반함. 해석이 필요함.

### 답변

**핵심 메커니즘: Pooled R²의 횡단면 성분 (Cross-sectional Component)**

Pooled R²는 두 가지 성분의 합으로 분해됩니다:

```
Pooled R² = 횡단면 성분 (어느 자산이 평균적으로 더 변동성이 높은가)
           + 시계열 성분 (각 자산의 변동성이 시간에 따라 언제 오르내리는가)
```

장기 예측에서 Pooled R²가 높은 것은 **시계열 타이밍 예측력이 뛰어나서가 아니라**, 자산군별 평균 변동성 수준 차이(USO >> TLT 등)를 모델이 잘 포착하기 때문입니다. 이 횡단면 성분은 장기로 갈수록 지배적이 됩니다.

**R²_Within(시계열 내 예측력만) vs Pooled R² 비교:**

| 기간 | Pooled R² | R²_Within | P-W 차이 |
|------|-----------|-----------|---------|
| 1d | 0.1176 | 0.0335 | +0.084 |
| 5d | 0.6249 | 0.2598 | +0.365 |
| 22d | 0.7974 | 0.3668 | +0.431 |
| 60d | 0.7853 | 0.1451 | +0.640 |
| 90d | 0.7753 | 0.0393 | +0.736 |
| 120d | 0.7683 | **-0.0912** | +0.860 |
| 180d | 0.7647 | **-0.1991** | +0.964 |
| 252d | 0.7342 | **-0.6995** | +1.434 |

**해석:** 120거래일 이상에서 R²_Within이 음수로 전환되는 것은, 모델이 각 자산 내 변동성 타이밍 예측에서는 단순 평균보다도 못함을 의미합니다. 높은 Pooled R²는 "어느 자산이 더 변동성이 크다"는 횡단면 순위 정보에서 비롯됩니다.

**따라서 올바른 해석:**
- 단기(1~90d): Pooled R²와 R²_Within이 모두 양수 → 시계열 타이밍 예측력 존재
- 장기(120d+): Pooled R²는 높으나 R²_Within 음수 → 시계열 타이밍 예측이 아닌 횡단면 수준 예측

장기 Pooled R²를 단기와 단순 비교하여 "장기 예측이 더 쉽다"로 해석하는 것은 오류입니다.

## 피드백 6. 비교 실험의 최신성

**피드백 요지:** 비교 대상 알고리즘이 다소 오래된 기법에 한정되어 있음.

### 답변

**현재 비교 모형군:**

| 유형 | 모형 | 최신성 |
|------|------|--------|
| 계량경제 | HAR-3, GARCH(1,1) | 금융 변동성 분야 표준 benchmark |
| 규제 선형 | Ridge, LASSO, ElasticNet | 고전적이나 변동성 예측에서 강력 |
| 트리·부스팅 | RF, **XGBoost (2016)**, **LightGBM (2017)** | 상대적으로 최신 |
| 딥러닝 | MLP, **BiLSTM-A** | 시계열 분야 주요 아키텍처 |
| 앙상블 | WEns (제안 모형) | |

**보완 방향:**

현재 ML 모형군(XGBoost, LightGBM, BiLSTM-A)이 이미 2016~2018년 이후 발표된 기법을 포함하고 있습니다. 한계로 지적될 수 있는 부분과 실현 가능성은 다음과 같습니다.

| 최신 기법 | 적용 가능성 | 비고 |
|----------|------------|------|
| CatBoost | 가능 | XGBoost와 유사 계열, 추가 시 비교값 제시 가능 |
| HAR-J (Jump) | 일별 데이터로 부분 근사 가능 | 틱 데이터 없어 완전 구현 불가 |
| HARQ (Realized Quarticity) | 불가 | 고빈도 데이터 필요 |
| Transformer (PatchTST 등) | 가능하나 표본 크기 제한 | 자산군별 소표본 문제 |

금융 변동성 예측 분야에서는 HAR과 GARCH가 현재도 주요 baseline으로 사용됩니다(Patton & Sheppard 2015; Bollerslev et al. 2018). 본 연구의 ML 비교군(RF, XGBoost, LightGBM, BiLSTM-A)은 해당 분야에서 최신에 속하는 기법들입니다.

## 피드백 7. 시계열 모델과의 비교 (Repeat baseline)

**피드백 요지:** 시계열 전용 모델과의 비교가 필요. 특히 Repeat(naive repeat) 기법과의 비교 권장.

### 답변

**Repeat baseline 추가 완료.**

리뷰어의 지적에 따라 **RepeatRV** baseline을 새로 추가하였습니다.

**정의:**
- **NaiveRV (기존):** 예측 기간 h에 맞춘 h-거래일 후방 이동평균 실현분산 (horizon-matched)
- **RepeatRV (신규):** 예측 기간에 무관하게 직전 22거래일 이동평균 RV를 고정 반복 사용 (horizon-agnostic)

**실험 결과 (Pooled R²):**

| 기간 | NaiveRV | RepeatRV | HAR-3 | WEns |
|------|---------|----------|-------|------|
| 1d | -0.566 | **-0.148** | 0.110 | **0.118** |
| 5d | 0.318 | **0.512** | 0.574 | **0.625** |
| 22d | 0.707 | 0.707 | 0.761 | **0.797** |
| 60d | **0.788** | 0.714 | 0.785 | 0.785 |
| 90d | 0.765 | 0.710 | **0.786** | 0.775 |
| 252d | 0.504 | 0.457 | 0.694 | **0.734** |

**주요 발견:**
1. **단기(1d, 5d)에서 RepeatRV > NaiveRV:** 단일 일별 제곱수익률보다 22d 이동평균이 훨씬 안정적.
2. **22d에서 두 baseline 동일:** 22d 창을 공유하므로 수치 일치.
3. **장기(60d+)에서 NaiveRV > RepeatRV:** horizon-matched 창이 장기 타깃에 더 적합.
4. **WEns는 모든 구간에서 두 Naive baseline 대비 우위:** 단기·중기에서 통계적으로 유의한 DM 검정 확인.

**HAR-3의 시계열 모형 지위:**

HAR-3(Corsi 2009)는 변동성의 장기기억(long memory) 특성을 이질적 자기상관 구조로 포착하는 계량경제학적 시계열 모형으로, 본 연구에서 시계열 전용 기준 모형을 대표합니다. HAR-3 대비 WEns의 우위는 22거래일 기준 DM 통계량 +5.391(p < 0.001)로 통계적으로 유의하게 확인됩니다(단, 60d 이상 장기에서는 유의하지 않음).
