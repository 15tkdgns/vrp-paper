## 4. Experiment B: DLinear

### 4.1 배경

Zeng et al. (2023) "Are Transformers Effective for Time Series Forecasting?", **AAAI 2023**.  
단순 선형 분해 모델(DLinear)이 Autoformer, FEDformer 등 복잡한 Transformer를 능가함을 보인 논문.  
핵심 주장: "시계열 예측에서 단순한 선형 모델이 강력하다."  

본 연구에 적합한 이유:  
- 구현이 매우 단순 (별도 패키지 불필요)  
- WEns의 피처 엔지니어링 없이 raw 시계열만으로 비교 가능  
- "단순 최신 기법 vs 정교한 피처 기반 앙상블" 대비 효과

### 4.2 모델 구조

```
입력: 직전 L일의 일별 log RV 시계열 [y_{t-L}, ..., y_{t-1}]
      (y_t = log(r²_t × 252), 22d 롤링이 아닌 1일 단위)

분해:
  Trend(t)    = MovingAverage(y, kernel_size=k)   # 저주파 성분
  Seasonal(t) = y - Trend(t)                       # 고주파 성분

예측:
  ŷ_trend    = W_trend × Trend     (선형 레이어)
  ŷ_seasonal = W_seasonal × Seasonal (선형 레이어)
  ŷ = ŷ_trend + ŷ_seasonal
```

**하이퍼파라미터 탐색:**
- `L` (lookback): {22, 63, 126, 252}
- `kernel_size` (이동평균 창): {5, 11, 25}

### 4.3 데이터 구조 차이

기존 WEns와 달리 DLinear/PatchTST는 **순차 입력** 방식:

```python
# 기존 WEns: 날짜 × 피처 행렬 (cross-sectional)
X_train.shape = (N_samples, 35)    # N = 전체 날짜 × 11 자산

# DLinear/PatchTST: 자산별 순차 시퀀스 (sequential)
X_train.shape = (N_samples, L, 1)  # N = 각 자산별 슬라이딩 윈도우 수
# → 자산별로 독립 학습 (per-asset fitting)
```

### 4.4 구현 코드 (핵심)

```python
import numpy as np
from sklearn.linear_model import LinearRegression

class DLinear:
    def __init__(self, lookback=126, kernel_size=25):
        self.L = lookback
        self.k = kernel_size
        self.w_trend = None
        self.w_season = None

    def _decompose(self, seq):
        # seq: (N, L) - N개 슬라이딩 윈도우, 각 길이 L
        pad = self.k // 2
        trend = np.array([
            np.convolve(seq[i], np.ones(self.k)/self.k, mode='same')
            for i in range(len(seq))
        ])
        seasonal = seq - trend
        return trend, seasonal

    def fit(self, seq_train, y_train):
        trend, seasonal = self._decompose(seq_train)
        self.w_trend   = LinearRegression().fit(trend, y_train)
        self.w_season  = LinearRegression().fit(seasonal, y_train)

    def predict(self, seq_test):
        trend, seasonal = self._decompose(seq_test)
        return self.w_trend.predict(trend) + self.w_season.predict(seasonal)
```

### 4.5 학습 데이터 구성 (per-asset, 자산별 독립)

```python
def make_sequences(log_rv_series, horizon, lookback):
    """자산별 1D log RV 시계열 → (슬라이딩 윈도우, 타깃) 쌍"""
    seqs, targets = [], []
    for t in range(lookback, len(log_rv_series) - horizon):
        seqs.append(log_rv_series[t-lookback:t])
        # 타깃: t+1 ~ t+h의 평균 (전방 RV)
        targets.append(np.mean(log_rv_series[t:t+horizon]))
    return np.array(seqs), np.array(targets)
```

## 5. Experiment C: PatchTST

### 5.1 배경

Nie et al. (2022) "A Time Series is Worth 64 Words: Long-term Forecasting with Transformers", **ICLR 2023**.  
시계열을 패치(patch) 단위로 분할하여 Transformer에 입력 → 긴 맥락 포착 + 연산 효율화.  
GitHub: [PatchTST](https://github.com/yuqinie98/PatchTST)  
Citation 수: 2000+ (2024년 기준 최인용 시계열 Transformer 논문 중 하나)

### 5.2 모델 구조

```
입력 시퀀스: L일 log RV (또는 일별 log return)
패치 분할: 패치 크기 P, stride S → N_patches = (L - P) / S + 1
Transformer Encoder: d_model, n_heads, n_layers
예측 헤드: Linear → h-step 예측
```

**하이퍼파라미터:**
- `patch_len`: {8, 16, 32}
- `stride`: {4, 8, 16}
- `d_model`: {64, 128}
- `n_heads`: {4, 8}
- `n_layers`: {2, 3}
- `dropout`: {0.1, 0.2}

## 9. 구현 시 핵심 주의사항

1. **데이터 분할 일관성**: DLinear/PatchTST의 학습/테스트 split은 WEns와 동일 날짜 기준 사용  
2. **Purge gap**: 순차 모델에도 동일하게 h-거래일 purge gap 적용 (슬라이딩 윈도우 경계)  
3. **평가 기준**: Pooled R²(11자산 전체 풀링) 동일하게 사용  
4. **학습 데이터 크기**: 자산별 ~1500~2000개 학습 샘플 → PatchTST 과적합 위험 있음  
   → Dropout 강화, 모델 크기 축소 (d_model=64, n_layers=2)로 대응  
5. **CatBoost**: `iterations` (= n_estimators)는 200~500 사이로 고정 후 탐색 대상 제외 권장
