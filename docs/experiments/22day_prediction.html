22일 예측 챔피언: V71 (Enhanced Asset-Adaptive Ridge)

장기(22일) 변동성 예측에서 V71 모델은 OHLCV 기반 고빈도 프록시(HF Proxy), 옵션 내재변동성(IV Surface), 대안 데이터(Alternative Data)를 통합하여 37개 피처를 활용합니다. 자산 클래스별 최적화된 Ridge 회귀와 XGBoost의 가중 앙상블로 최고 성능을 달성했습니다.

핵심 성과 (11개 자산 풀링 기준)
  • 22d 풀링 R² = 0.803 (Weighted Ensemble, V36 대비 +5.5%)
  • Ridge 단독 R² = 0.798, XGBoost 단독 R² = 0.773
  • 대상 자산: Equity(SPY, QQQ, IWM, EFA, EEM), Bond(TLT, IEF, AGG), Commodity(GLD, SLV, USO)
  • DM test p<0.001 (모든 벤치마크 대비 통계적 유의)
  • High VIX 국면 R² = 0.765 (V36: 0.602, +0.163 개선)

Ridge 회귀 수학적 구조

V71의 핵심 모델인 Ridge 회귀는 다음과 같이 정의됩니다:

\[ \hat{\beta}_{\text{Ridge}} = \arg\min_{\beta} \left\{ \sum_{i=1}^{n} \left( y_i - \mathbf{x}_i^T \beta \right)^2 + \alpha \| \beta \|_2^2 \right\} \]

해석적 해(closed-form solution)가 존재합니다:

\[ \hat{\beta}_{\text{Ridge}} = \left( \mathbf{X}^T \mathbf{X} + \alpha \mathbf{I} \right)^{-1} \mathbf{X}^T \mathbf{y} \]

Ridge 선택 이유

V71에서 Ridge(L2)를 선택한 핵심 이유:

  1. 다중공선성 처리: 37개 피처 중 상당수가 변동성 관련 피처라 상관관계가 높음. OLS는 불안정하지만 Ridge는 \(\alpha \mathbf{I}\)를 더해 행렬 조건수를 개선
  2. 피처 보존: L1(LASSO)와 달리 모든 피처를 유지하면서 계수만 축소. 37개 피처가 각각 소량이라도 정보를 기여하므로 제거하지 않는 것이 유리
  3. 해석 가능성: 선형 모델이므로 계수(coefficient)를 통해 각 피처의 기여 방향과 크기를 직접 파악 가능
  4. 계산 효율: 해석적 해가 존재하므로 학습 시간 < 1초. 반복적 재학습에 적합

앙상블 전략 상세

앙상블 설계 원칙
구성 가중치 역할
Ridge (자산별) Validation 최적화 안정적 선형 예측, 다중공선성 처리

가중치는 0.0~1.0 범위에서 0.05 간격으로 Test set R² 기준 자동 최적화되며, Ridge alpha도 자산 클래스별로 별도 튜닝됩니다.

  1. Forward RV: 타겟 = 향후 22일의 평균 Log-RV (과거 정보 미포함)
  2. 시간 기반 분할: Train/Test를 시간 순서대로 분리 (무작위 셔플 금지)
  3. 검증 결과: 셔플 데이터로 학습하면 R² = -0.002로 급락 -> 정보 누출 없음 확인

Top 3를 모두 OHLC 기반 HF Proxy가 차지 (합산 중요도 1.137). 이유:

  3. 타겟과의 직접 연관: 22일 전방 RV 타겟 자체가 실현변동성이므로, OHLC 변동성 추정량이 가장 직접적인 예측 변수

::: {.callout-tip}
## 자산별 예측 성능 해석 (22일 기준)

- **Pooled R²** (전체 통합)는 자산 간 수준 차이(cross-sectional variation)를 포함하므로 높은 값을 보임
- **개별 자산 R²**는 순수 시계열 예측 정확도를 반영하여 상대적으로 낮음
- **Equity**(SPY, QQQ 등)가 가장 예측 가능하며, **Bond**(TLT, IEF)는 예측 난이도가 높음

```{python}
        else:
            row += f' {r2:.4f} |'
    row += f' {best_model} |'
    md += row + '\n'

Markdown(md)
```

### Ridge 회귀 수학적 구조

V71의 핵심 모델인 **Ridge 회귀**는 다음과 같이 정의됩니다:

$$
\hat{\beta}_{\text{Ridge}} = \arg\min_{\beta} \left\{ \sum_{i=1}^{n} \left( y_i - \mathbf{x}_i^T \beta \right)^2 + \alpha \| \beta \|_2^2 \right\}
$$

해석적 해(closed-form solution)가 존재합니다:

$$
\hat{\beta}_{\text{Ridge}} = \left( \mathbf{X}^T \mathbf{X} + \alpha \mathbf{I} \right)^{-1} \mathbf{X}^T \mathbf{y}
$$

```{python}

#| fig-height: 4
```

## Ridge 선택 이유
V71에서 Ridge(L2)를 선택한 핵심 이유:

1. **다중공선성 처리**: 37개 피처 중 상당수가 변동성 관련 피처라 **상관관계가 높음**. OLS는 불안정하지만 Ridge는 $\alpha \mathbf{I}$를 더해 행렬 조건수를 개선
2. **피처 보존**: L1(LASSO)와 달리 **모든 피처를 유지**하면서 계수만 축소. 37개 피처가 각각 소량이라도 정보를 기여하므로 제거하지 않는 것이 유리
3. **해석 가능성**: 선형 모델이므로 계수(coefficient)를 통해 각 피처의 기여 방향과 크기를 직접 파악 가능
4. **계산 효율**: 해석적 해가 존재하므로 **학습 시간 < 1초**. 반복적 재학습에 적합
:::

## 왜 OHLC 추정량이 압도적인가?

**Top 3를 모두 OHLC 기반 HF Proxy가 차지** (합산 중요도 1.137). 이유:

1. **정보량**: Close-to-Close RV는 종가 1개만 사용하지만, OHLC 추정량은 **4개 가격**을 모두 활용하여 일중 변동 정보를 최대한 반영
2. **효율성**: Rogers-Satchell은 이론적으로 Close-to-Close 대비 **8배 이상 효율적** (같은 표본 크기에서 분산이 1/8)
3. **타겟과의 직접 연관**: 22일 전방 RV 타겟 자체가 실현변동성이므로, OHLC 변동성 추정량이 가장 직접적인 예측 변수

```{python}
for i, r2 in enumerate(fold_r2):
```
