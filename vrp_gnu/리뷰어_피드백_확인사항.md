## 4. 결측치 처리(전방 채움) 근거

### 4.1 전방 채움을 선택한 이유

`v71_robustness_academic.py`의 `build_dataset_robust()` 함수와 모든 실험 스크립트에서 `raw.ffill()`을 일관되게 사용합니다. 선택 이유는 복합적입니다:

1. **미래 정보 누출 방지**: 선형 보간(linear interpolation)은 미래 시점의 값을 참조하여 결측을 채우므로, 시계열 예측에서 데이터 누출 위험이 있습니다. 전방 채움은 과거 관측값만 사용하므로 이 문제가 없습니다.
2. **시계열 연속성 유지**: 결측 행을 삭제하면 롤링 윈도우(5일, 22일) 계산에서 시계열 연속성이 깨지고, GARCH 적합 등에 영향을 줍니다.
3. **표본 손실 최소화**: ffill 사용 시 41,261 표본, 미사용 시 38,489 표본으로 약 6.7%의 표본이 보존됩니다.

### 4.2 비교 실험 결과 (ffill vs 삭제)

`v71_robustness_academic_results.json`에서 확인한 비교 실험 결과:

| 설정 | 표본 수 | 풀링 R^2 | 중위 R^2 | RMSE |

| **전방 채움 사용** | 41,261 | 0.782 | 0.123 | 0.488 |

## 6. 2단계 접근의 정확한 설계 의도

### 6.1 코드에서 확인된 구조

`tmp_calc_wens_multi_horizon.py` 라인 50~53:
```python
def forward_rv(ret_sq, horizon):
    cs = ret_sq.cumsum()
    fwd_mean = (cs.shift(-horizon) - cs) / horizon
    return np.log(fwd_mean * 252 + 1e-12)
```

모델은 `log(RV)`를 직접 예측하고, VRP는 사후적으로 `VRP = IV^2 - RV_hat`로 산출합니다.

### 6.2 설계 의도

논문 3.6절과 코드를 종합하면, "중첩 방지"의 정확한 의미는 다음과 같습니다:

1. **피처-타겟 정보 중첩 방지**: VRP를 직접 타겟으로 학습시키면, 타겟(VRP = IV - RV)에 이미 포함된 IV 정보가 피처(VIX, VIX3M 등)에도 동일하게 존재합니다. 즉 **동일 정보(IV)가 피처와 타겟 양쪽에 중복 투입**됩니다. 모델이 단순히 "IV를 그대로 전달"하는 지름길을 학습할 위험이 있습니다.
2. **RV 예측의 순수성 확보**: RV만 예측 대상으로 삼으면, 모델은 순수하게 "미래 변동성이 얼마나 될까?"를 학습합니다. 관측 가능한 IV는 예측 시점에 이미 알려진 값이므로, VRP 산출 시 추가 예측이 필요 없습니다.
3. **해석 가능성**: RV 예측 오차와 VRP 예측 오차가 부호만 반대이고 절대 크기가 동일하므로(논문 4.1.1절), 성능 비교가 단순화됩니다.

**한 문장 요약**: "VRP를 직접 예측하면 피처(VIX 기간구조)와 타겟(IV-RV)에 IV가 중복 투입되어 편향된 학습이 발생할 수 있으므로, RV만을 순수 예측 대상으로 분리하고 IV는 사후적으로 결합하는 2단계 설계를 채택하였다."

**한 문장 요약**: "기본 선형 구조에서는 8피처면 충분하지만, 자산군별 적응적 규제와 앙상블 결합이 추가될 때 37피처의 여분 정보(특히 고변동성 국면의 IV Surface)가 추가 개선을 제공한다."

## 10. USO(원유 ETF) 롤오버 효과 설명

논문 3.1절에서 이미 서술된 내용의 논리:

- USO는 WTI 원유 선물 기반 ETF로, 콘탱고(contango) 구간에서는 선물 롤오버 시 고가 매수가 발생하여 **가격 하방 편향**이 존재합니다. 백워데이션(backwardation)에서는 반대입니다.
- 이 롤오버 비용은 USO의 가격 변동에 포함되어 있으므로, USO로 계산한 **실현 변동성은 "순수 원유 현물 변동성"이 아니라 "선물 롤 구조를 포함한 투자자 관점의 변동성"**입니다.
- 이는 본 연구가 투자자의 실제 포트폴리오 관점에서 VRP를 분석하는 것과 정합적이며, 현물 변동성과의 괴리를 한계로 인정하면 충분합니다.
